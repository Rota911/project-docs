# LS Core

LS Core is the central framework for all server resources. It provides a modular architecture with lazy-loaded modules, a shared cache system, and standardized APIs for client-server communication.

## Requirements

- Lua 5.4 (`lua54 'yes'` in manifest)
- `ls_core` must be started before any dependent resource

## Setup

Add the init script to your resource manifest:

```lua
-- fxmanifest.lua
fx_version 'cerulean'
game 'gta5'
lua54 'yes'

shared_script '@ls_core/init.lua'
```

This exposes three globals to your resource:

| Global | Description |
|--------|-------------|
| `LS` | Module loader and core API accessor |
| `cache` | Auto-updating value cache with event hooks |
| `require` | Module require system (replaces native) |

## LS Object

The `LS` object provides access to all modules. Modules are lazy-loaded on first access.

```lua
---@type table
LS.Logger      -- Logger module
LS.Callback    -- Callback module
LS.Anim        -- Animation module
LS.Blips       -- Blips module
LS.Zones       -- Zones module
LS.UI          -- UI module
LS.Table       -- Table extensions
LS.Math        -- Math extensions
LS.Cron        -- Cron scheduler (server)
LS.Keybinds    -- Keybind manager (client)
LS.Raycast     -- Raycast utility (client)
LS.Peds        -- Ped manager (client)
LS.Objects     -- Object manager
LS.Marker      -- Marker module (client)
LS.Camera      -- Camera module (client)
LS.Grid        -- Spatial grid system
LS.Interact    -- Interaction system (client)
```

## Cache

The `cache` table stores frequently accessed values and updates them automatically via events.

### Built-in Values (Client)

```lua
---@type number
cache.playerId    -- PlayerId()
---@type number
cache.serverId    -- GetPlayerServerId()
---@type number
cache.ped         -- PlayerPedId() (auto-updated)
---@type number|false
cache.vehicle     -- Current vehicle (auto-updated)
---@type number|false
cache.seat        -- Current vehicle seat (auto-updated)
---@type number|false
cache.weapon      -- Current weapon hash (auto-updated)
```

### Built-in Values (Shared)

```lua
---@type string
cache.resource    -- GetCurrentResourceName()
---@type string
cache.game        -- GetGameName()
```

### Custom Caching

```lua
---@generic T
---@param key string
---@param func fun(): T
---@param timeout? number  -- auto-clear after N ms
---@return T
local value = cache('myKey', function()
    return expensiveComputation()
end, 5000)
```

### Cache Change Listener

```lua
LS:OnCache('ped', function(newValue, oldValue)
    -- fires when cache.ped changes
end)
```

## Utility Functions

### SetInterval / ClearInterval

```lua
---@param callback function
---@param interval? number  -- ms, defaults to 0
---@return number id
local id = SetInterval(function()
    -- runs every 5 seconds
end, 5000)

ClearInterval(id)
```

## Module Registration

External resources can register themselves as LS modules, making them accessible via `LS.ModuleName` in any resource.

```lua
---@param moduleName string
---@param resourceName string
---@param path? string
---@return boolean success
---@return string|nil message
LS:RegisterModule('MyModule', 'my_resource', 'optional/path')
```

Once registered, any resource can access it:

```lua
LS.MyModule:DoSomething()
```

## Client Context

On the client, `LS.PlayerData` and `LS.CharacterData` are automatically populated and kept in sync:

```lua
---@type table
LS.PlayerData       -- { identifier, discord, adminGroup, ... }
---@type table
LS.CharacterData    -- { charId, firstName, lastName, fullName, groups, ... }
```

These update automatically via the following events:

| Event | Trigger |
|-------|---------|
| `Core:Client:PlayerJoined` | Player data initialized |
| `Core:Client:CharacterLoaded` | Character selected |
| `Core:Client:CharacterLogout` | Character deselected |
| `Core:Client:PlayerDataChanged` | Player field updated |
| `Core:Client:CharacterDataChanged` | Character field updated |

## Server Context

Server-side utility functions available globally:

```lua
---@param poolName "CPed"|"CObject"|"CVehicle"
---@return number[]
GetGamePool(poolName)

---@return number[]
GetActivePlayers()
```

## Common Errors

| Error | Cause | Fix |
|-------|-------|-----|
| `Lua 5.4 must be enabled` | Missing Lua 5.4 flag | Add `lua54 'yes'` to fxmanifest |
| `ls_core must be started before this resource` | Boot order | Ensure `ls_core` starts first in server.cfg |
| `Cannot load ls_core more than once` | Duplicate init | Remove duplicate `@ls_core/init.lua` entries |
