# Streaming Requests

Client-side asset loading helpers. All functions yield the current thread until the asset is loaded or timeout is reached. Access via `LS:Request*`.

## RequestModel

```lua
---@param model number|string -- hash or name (auto-hashed if string)
---@param timeout? number -- default 10000ms
---@return number model
LS:RequestModel(model, timeout)
```

Errors if the model is invalid (`IsModelValid` / `IsModelInCdimage`).

## RequestAnimDict

```lua
---@param animDict string
---@param timeout? number -- default 10000ms
---@return string animDict
LS:RequestAnimDict(animDict, timeout)
```

Errors if the dictionary does not exist (`DoesAnimDictExist`).

## RequestWeaponAsset

```lua
---@param weaponType string|number
---@param timeout? number -- default 10000ms
---@param weaponResourceFlags? WeaponResourceFlags -- default 31 (WRF_REQUEST_ALL_ANIMS)
---@param extraWeaponComponentFlags? ExtraWeaponComponentFlags -- default 0
---@return string|number weaponType
LS:RequestWeaponAsset(weaponType, timeout, weaponResourceFlags, extraWeaponComponentFlags)
```

## StreamingRequest (Internal)

All `Request*` functions delegate to this internal helper. Not typically called directly.

```lua
---@param request function -- native request function
---@param hasLoaded function -- native check function
---@param assetType string -- for error messages
---@param asset any
---@param timeout? number
---@param ... any -- extra args forwarded to request()
LS:StreamingRequest(request, hasLoaded, assetType, asset, timeout, ...)
```

If the asset is already loaded, returns immediately. Otherwise calls `request(asset, ...)` and yields via `LS:WaitFor` until `hasLoaded(asset)` returns true.

## Behavior

- All functions return immediately if the asset is already loaded
- On timeout, an error is raised with a descriptive message
- Default timeout is 10000ms for most assets, 30000ms for audio banks, 1000ms for scaleforms
