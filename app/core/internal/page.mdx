# Server API

Internal class systems that manage players, characters, groups, and permissions on the server side.

## Architecture

```
core/server/
  player/
    loading.lua            -- Connection and join handling
    functions/
      getPlayer.lua        -- GetPlayer() global
      getCharacter.lua     -- GetCharacter() global
      getCharacters.lua    -- GetCharacters() global
    modules/
      playerClass.lua      -- Player class constructor and methods
      characterClass.lua   -- Character class constructor and methods
      database.lua         -- Player/character DB queries
  group/
    main.lua               -- Group loading and init
    modules/
      class.lua            -- Group class constructor and methods
      database.lua         -- Group DB queries
  acl/
    server.lua             -- ACE/Principal helpers
  admin/
    main.lua               -- Admin system init
    modules/
      command.lua          -- LS:Command registration
      function.lua         -- LS:RegisterFunction / LS:ExecuteFunction
  routing/
    main.lua               -- Routing bucket management
  anim/
    main.lua               -- Server-side animation prop sync
```

## Lookup Functions

These are globally available on the server.

### GetPlayer

```lua
---@param identifier number|string  -- source, playerId, discord ID, or license2
---@return Player|nil
local player = GetPlayer(identifier)
```

Identifier type is resolved automatically:

| String Length | Resolved As |
|--------------|-------------|
| 1-4 digits | source |
| 5-17 digits | playerId |
| 18 digits | discord |
| Contains letters | license2 |

### GetCharacter

```lua
---@param identifier number|string  -- source (1-4 digits) or charId
---@return Character|nil
local character = GetCharacter(identifier)
```

### Storage

All live data is stored in `Private` tables:

```lua
Private.Players     ---@type table<number, Player>     -- keyed by playerId
Private.Characters  ---@type table<number, Character>  -- keyed by charId
Private.Groups      ---@type table<string, Group>      -- keyed by group id string
Private.GroupTypes   ---@type table<string, table>      -- group type configs
```

## ACL

Wrappers around FiveM's ACE/Principal system.

```lua
---@param principal number|string  -- player source or principal string
---@param ace string
---@param allow? boolean  -- defaults to true
LS:AddAce(principal, ace, allow)
LS:RemoveAce(principal, ace, allow)

---@param child number|string
---@param parent string
LS:AddPrincipal(child, parent)
LS:RemovePrincipal(child, parent)
```

A callback is registered for client-side permission checks:

```lua
-- Server (auto-registered)
LS.Callback:Register('Core:Server:CheckPlayerAce', function(source, command)
    return IsPlayerAceAllowed(source, command)
end)
```

## Events

### Server Events

| Event | Parameters | Description |
|-------|-----------|-------------|
| `Core:Server:CharacterLoaded` | `source, charId` | Character selected and loaded |
| `Core:Server:CharacterLogout` | `source, charId` | Character unloaded |
| `Core:Server:CharacterCreated` | `source, characterData` | New character created |
| `Core:Server:CharacterGroupChanged` | `source, charId, group, rank, allGroups` | Character group updated |
| `Core:Server:PlayerLeft` | `source, reason` | Player disconnected |
| `Core:Server:ScriptRestart` | none | ls_core restarted |

### Client Events

| Event | Parameters | Description |
|-------|-----------|-------------|
| `Core:Client:CharacterLoaded` | `characterData` | Character data sent to client |
| `Core:Client:CharacterLogout` | none | Character unloaded on client |
| `Core:Client:CharacterCreated` | `characterData` | Character created notification |
| `Core:Client:PlayerJoined` | `_, playerData` | Player data initialized |
| `Core:Client:PlayerDataChanged` | `key, value` | Player field updated |
| `Core:Client:CharacterDataChanged` | `key, value` | Character field updated |
| `Core:Client:CharacterGroupChanged` | `group, rank, allGroups` | Group change notification |
| `Core:Client:ScriptRestart` | none | ls_core restarted |

## Auto-Save

Characters are automatically saved every 5 minutes via `LS.Cron`. A save also runs on `txAdmin:events:serverShuttingDown` and 10 seconds before a scheduled restart.
