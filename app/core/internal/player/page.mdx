# Player

Server-side class representing a connected player. Manages authentication state, admin permissions, duty, points, and character selection.

## Retrieving a Player

```lua
---@param identifier number|string
---@return Player|nil
local player = GetPlayer(identifier)
```

Identifier resolution is automatic based on input:

| Input | Resolved As |
|-------|-------------|
| 1-4 digit number | source |
| 5-17 digit number | playerId |
| 18 digit number | discord |
| String | license2 |

Also available as `LS:GetPlayer(identifier)`.

## Class Definition

```lua
---@class Player
---@field source number
---@field identifier string          -- license2
---@field discord string             -- discord ID
---@field playerId number            -- database player ID
---@field adminGroup string          -- 'user', 'moderator', 'admin', 'headadmin', 'dev', 'owner'
---@field points number
---@field duty boolean               -- admin duty active
---@field characters table<number>   -- list of charId values
---@field selectedCharacter number|nil
---@field isLoggedIn boolean         -- true when a character is active
```

## Methods

### Event

Triggers a client event on this player.

```lua
---@param eventName string
---@param ... any
player:Event(eventName, ...)
```

### SetAsLoggedIn

Sets the login state and syncs `isLoggedIn` to the player's state bag.

```lua
---@param state boolean
player:SetAsLoggedIn(state)
```

### SetGroup

Sets the player's admin group. Persists to database unless `temporary` is true.

```lua
---@param group string
---@param temporary? boolean
---@return boolean success
---@return string message
player:SetGroup(group, temporary)
```

### GetGroup

```lua
---@return string
player:GetGroup()
```

### SetDuty

Toggles or sets admin duty. Manages ACE principals and the `isAdmin` state bag.

```lua
---@param state? boolean  -- nil to toggle
player:SetDuty(state)
```

### GetDuty

```lua
---@return boolean
player:GetDuty()
```

### GetPoints

```lua
---@return number
player:GetPoints()
```

### SetPoints

Overwrites the point value. Persists to database and notifies client.

```lua
---@param value number
player:SetPoints(value)
```

### AddPoints

Adds to the current point value.

```lua
---@param value number
player:AddPoints(value)
```

### RemovePoints

Subtracts from the current point value. Floors at 0.

```lua
---@param value number
player:RemovePoints(value)
```

### CreateCharacter

Creates a new character. Maximum 3 per player. Saves appearance via `ls_appearance`, then calls `SetActiveCharacter`.

```lua
---@param characterData table
---@return boolean success
---@return string reason  -- 'success_create_character' | 'maximum_character' | 'cannot_save_character' | 'cannot_save_appearance'
player:CreateCharacter(characterData)
```

### SetActiveCharacter

Loads a character from the database, constructs the Character object, stores it in `Private.Characters`, and fires load events.

```lua
---@param characterId number
---@return boolean
player:SetActiveCharacter(characterId)
```

Returns `false` if the player is already logged in or the character data cannot be fetched.

### GetActiveCharacter

```lua
---@return Character|nil
player:GetActiveCharacter()
```

Returns `nil` if the player is not logged in.

### UpdateSource

Updates the stored source after a reconnect.

```lua
---@param newSource number
player:UpdateSource(newSource)
```

## Constructor

The Player object is constructed internally during `playerConnecting`. It resolves or creates the database record, loads characters, admin group, and points.

```lua
-- Internal call (core/server/player/loading.lua)
local player = require 'core.server.player.modules.playerClass'(playerSrc, identifiers)
Private.Players[player.playerId] = player
```

The constructor also calls `LS:AddPrincipal(source, 'user')` to set base permissions.

## State Bag

| Key | Replicated | Description |
|-----|-----------|-------------|
| `playerId` | No | Database player ID |
| `isLoggedIn` | Yes | Whether a character is active |
| `isAdmin` | Yes | Set when duty is on, cleared when off |

## Events Fired

| Event | Side | When |
|-------|------|------|
| `Core:Client:PlayerDataChanged` | Client | Any field change (`key, value`) |
| `Core:Client:CharacterLoaded` | Client | Character selected |
| `Core:Client:CharacterCreated` | Client | Character created |
| `Core:Server:CharacterLoaded` | Server | Character selected |
| `Core:Server:CharacterCreated` | Server | Character created |
