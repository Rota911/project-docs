## Retrieving a Character

```lua
---@param identifier number|string  -- source (1-4 digits) or charId (5+ digits)
---@return Character|nil
local character = GetCharacter(identifier)
```

Also available as `LS:GetCharacter(identifier)`.

Characters are stored in `Private.Characters` keyed by `charId`.

## Class Definition

```lua
---@class Character
---@field source number            -- player source
---@field charId number            -- character database ID
---@field playerId number          -- owning player's database ID
---@field stateId string           -- 7-char alphanumeric state ID
---@field firstName string
---@field middleName string
---@field lastName string
---@field fullName string          -- computed from first/middle/last
---@field dateOfBirth string
---@field nationality string
---@field gender string
---@field groups table             -- { [groupId] = rankId, ... }
---@field specialization string    -- 'citizen', 'criminal', etc.
---@field position vector4
---@field statuses table
---@field skills table
```

## Methods

### Event

Triggers a client event on the character's owning player.

```lua
---@param eventName string
---@param ... any
character:Event(eventName, ...)
```

### GetPed

Returns the server-side ped handle for this character.

```lua
---@return number
character:GetPed()
```

### Save

Persists current position, statuses, groups, and skills to the database.

```lua
---@return boolean success
character:Save()
```

## Data

### SetData

Sets a value on the character and notifies the client. Optionally syncs to the player's state bag.

```lua
---@param key string
---@param value any
---@param state? boolean       -- also set on Player state bag
---@param replicated? boolean  -- replicate state to all clients
character:SetData(key, value, state, replicated)
```

### GetData

```lua
---@param key string
---@return any
character:GetData(key)
```

## Coordinates

### SetCoords

Teleports the character. Updates `Private.Characters` position unless `noSync` is true.

```lua
---@param newCoords vector3|vector4|table
---@param noSync? boolean  -- if true, does not update stored position
character:SetCoords(newCoords, noSync)
```

Accepts `vector3`, `vector4`, or a table with `x, y, z` (and optional `w`/`heading`).

### GetCoords

Returns current coordinates from the ped entity. Falls back to stored position if the ped does not exist.

```lua
---@param _type? "vector2"|"vector3"|"vector4"|"table"  -- defaults to "vector4"
---@return vector2|vector3|vector4|table
character:GetCoords(_type)
```

## Groups

### SetGroup

Adds, changes, or removes a group on this character. Delegates to `Group:AddMember` / `Group:RemoveMember` for validation and limit checks.

```lua
---@param group string       -- group ID
---@param rank? string       -- rank ID, nil to remove
---@param silent? boolean    -- suppress events and notifications
---@return boolean success
---@return string|nil reason
character:SetGroup(group, rank, silent)
```

### HasGroup

```lua
---@param group string
---@param rank? string  -- nil to check membership only
---@return boolean
character:HasGroup(group, rank)
```

### HasGroupType

Checks whether the character is a member of any group with the given type.

```lua
---@param groupType string  -- e.g. 'illegal', 'state', 'eatery'
---@return boolean
character:HasGroupType(groupType)
```

## Routing

### GetRoute

```lua
---@return number|nil  -- current routing bucket, nil if unset
character:GetRoute()
```

### SetRoute

Sets the character's routing bucket. Also moves attached entities.

```lua
---@param route number
---@param forced? boolean  -- set even if already in this route
---@return boolean
character:SetRoute(route, forced)
```

Fires `Routing:Client:NewRoute` on the client and sets `currentRoute` on the player state bag.

### GlobalRoute

Shortcut for `character:SetRoute(0, true)`.

```lua
---@return boolean
character:GlobalRoute()
```

## State Bag

| Key | Replicated | Description |
|-----|-----------|-------------|
| `charId` | Yes | Character database ID |
| `name` | Yes | Full name |
| `stateId` | Yes | 7-char state identifier |
| `currentRoute` | No | Current routing bucket |

## Events Fired

| Event | Side | When |
|-------|------|------|
| `Core:Client:CharacterDataChanged` | Client | `SetData`, `SetCoords`, group changes |
| `Core:Client:CharacterGroupChanged` | Client | Group added/removed/changed |
| `Core:Server:CharacterGroupChanged` | Server | Group added/removed/changed |
| `Core:Client:CharacterLogout` | Client | Character unloaded |
| `Core:Server:CharacterLogout` | Server | Character unloaded |
| `Core:Client:FadeOut` | Client | Screen fade during unload |
| `Routing:Client:NewRoute` | Client | Routing bucket changed |
