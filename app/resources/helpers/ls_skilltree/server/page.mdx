# Server

## Exports

### GetClass

Returns a `SkillTreeManagerProxy` — the primary way to interact with the skilltree system from an external resource.
Register your trees inside the callback passed to `RegisterTrees`, which is fired automatically after `Skilltree:RefreshSkilltrees`.

```lua
AddEventHandler("Skilltree:RefreshSkilltrees", function()
    exports["ls_skilltree"]:RegisterTrees(function(ST)
        local tree = ST:CreateAndRegister({
            id       = "mining",
            name     = "Mining",
            resource = GetCurrentResourceName(),
            baseXp   = 400,
            xpGrowth = 1.2,
            nodes    = { ... }
        })
    end)
end)
```

### RegisterTrees

Calls the provided factory function with a ready `SkillTreeManagerProxy`.
Handles proxy resolution internally — no manual `GetClass` or `Wait` needed.

```lua
---@param fn fun(ST: SkillTreeManagerProxy)
exports["ls_skilltree"]:RegisterTrees(function(ST)
    ST:CreateAndRegister({ id = "combat", ... })
end)
```

### CreateAndRegisterTree

Flat convenience export. Creates and registers a tree in one call without needing a proxy.

```lua
---@param data table
---@return boolean
local ok = exports["ls_skilltree"]:CreateAndRegisterTree({
    id       = "farming",
    name     = "Farming",
    baseXp   = 300,
    xpGrowth = 1.1,
    nodes    = { ... }
})
```

## SkillTreeManagerProxy

Obtained via `GetClass()` or injected by `RegisterTrees`. All methods below are available on the proxy.

### ST:Create

Creates a `SkillTreeInstance` and tracks it under the calling resource. Does **not** register it — call `:Register()` on the returned instance or use `ST:CreateAndRegister`.

```lua
---@param data table
---@return SkillTreeInstanceProxy|nil
local tree = ST:Create({
    id       = "combat",
    name     = "Combat",
    baseXp   = 500,
    xpGrowth = 1.15,
    nodes    = {
        ["basics"] = {
            preRequisites = {},
            children      = { "advanced" },
            lock          = {},
            unlocked      = false,
            label         = "Basic Combat",
            description   = "Fundamentals of combat.",
            cost          = 1,
        },
        ["advanced"] = {
            preRequisites = { "basics" },
            children      = {},
            lock          = {},
            unlocked      = false,
            label         = "Advanced Combat",
            description   = "Master-level techniques.",
            cost          = 2,
        },
    }
})
```

### ST:CreateAndRegister

Creates and immediately registers the tree. Equivalent to `ST:Create` followed by `tree:Register()`.

```lua
---@param data table
---@return SkillTreeInstanceProxy|nil
local tree = ST:CreateAndRegister({ id = "mining", ... })
```

## Instance Functions

Obtained from `ST:Create`, `ST:CreateAndRegister`, or `ST:Get`.

Registers the tree into the manager and notifies all connected clients.
Also initialises skill data for any already-loaded characters.

```lua

```

### tree:CanUnlockNode

Validates all conditions without making any changes. Useful for pre-flight checks.

```lua
---@param charId number
---@param nodeId string
---@return boolean canUnlock
---@return string|nil reason
local canUnlock, reason = tree:CanUnlockNode(charId, "ore_sense")
if not canUnlock then
    print("Cannot unlock:", reason)
end
```

### tree:UnlockNode

Deducts the node cost from the character's points and marks the node as unlocked.

```lua
---@param charId number
---@param nodeId string
---@return boolean success
---@return string|nil reason
local success, reason = tree:UnlockNode(charId, "ore_sense")
```

| Reason | Description |
|---|---|
| `invalid_node` | Node does not exist in the tree |
| `already_unlocked` | Node is already unlocked |
| `missing_prerequisite` | All required prerequisite nodes must be unlocked first |
| `missing_any_prerequisite` | At least one node from `anyPreRequisites` must be unlocked |
| `not_enough_points` | Character does not have enough skill points |
| `locked_by_other_node` | A mutually exclusive node is already unlocked |
| `no_character` | Character could not be found |

### tree:Respec

Refunds all spent points and clears `unlockedNodes`.

```lua
---@param charId number
---@return boolean success
---@return number refundedPoints
local success, refunded = tree:Respec(charId)
print(("Refunded %d points"):format(refunded))
```

### tree:GetAvailableNodes

Returns all node IDs the character can currently unlock (passes all conditions).

```lua
---@param charId number
---@return string[]
local available = tree:GetAvailableNodes(charId)
for _, nodeId in ipairs(available) do
    print("Can unlock:", nodeId)
end
```

## Events

### Server

| Event | Parameters | Description |
|---|---|---|
| `Skilltree:RefreshSkilltrees` | — | Fired by `ls_skilltree` after all default trees are registered. External resources should listen to this to register their own trees. |
| `SkillTree:Server:Register` | `treeId: string` | Fired when any tree is registered into the manager |
| `SkillTree:Server:Remove` | `treeId: string` | Fired when a tree is removed from the manager |

---@class SkillNode
---@field preRequisites string[]     All must be unlocked before this node is available

---@field label string
---@field description string
